webpack

注意事项

1 . 在使用webpack-dev-server的时候不要使用extract-text-webpack-plugin，它会把css独立成一个文件，这时候修改css就不能刷新浏览器，不能实时更新了。
2 . webpack在解析文件时候会通过3种路径寻找文件，一种是绝对路径，一种是相对路径，还有一种就是通过module，如果找到的是一个文件夹，就会查看该文件夹里面有没有package.json文件，如果有就依照其配置，如果没有就会按照resolve.mainFiles配置选项制定的文件名进行匹配。

各个关键词

extry: 单个入口语法，webpack从这个文件开始读取信息,可以传一个数组，会生成多个独立分离的依赖图表。

output: 控制webpack如何向硬盘写入编译文件，虽然extry可以有多个，但是output只能有一个

loaders: loader用于对模块的源代码进行转换，可以在require或者import模块时预处理文件，类似于gulp中的task

插件plugins：webpack插件是一个具有apply属性的javascript对象

markdown语法使用

概诉

  • Markdown是一种书写的格式，语法比较简单，用它编写的 纯文本可以转换成有效的XHTML文档，非常适合用来记录笔记、博客。熟练掌握它即可以让页面美观，也能大大提高编写效率。但在编写过程中需要注意大部分有符号的地方必需要空格才能生效。

段落和换行

  1. 段落的前后必须是空行,空行指的是行内什么都没有，或者只有空白符（空格或制表符），如果相邻2行文本中间没有空行就会被显示在一行内。
  2. 如果需要在段落内换行，可以在前一行的末尾加入2个空格，然后换行编写其他内容
  3. Markdown多数区块都需要在2个空行之间。

标题

  • Markdown支持2中形式的标题语法
  • 第一种：通过=(最高阶标题) -(第二阶标题)
  • 效果如下：

this is an H1

this is an H2

  • 第二种：通过输入1-6个#，对应的就是H1-H6,在标题后面可以添加#，也可以不添加#
  • 效果如下

this is an H1

this is an H2

区块引用

  • 每行前面加上>，段落的第一行前面可以添加>,区块引用可以嵌套，也可以引用标题、列表、代码区块等。
