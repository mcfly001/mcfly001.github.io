1.本次修改还没git add 这时候可以用git checkout –file就可以了
2.本次已经add或者commit但是没有推送到远程 这时候可以返回之前的版本

  • 2.1如果是add没有commit，这时候git reflow 选择最新的版本的commit Id 然后git reset –hard xx
  • 2.2如果commit但是没有推送到远程，这时候可以用也是和上一个方法一样，但是选择的commit id应该是上一个的，而不是最新的

情景六： git pull的时候refusing to merge unrelated histories,这是因为直接git init是没有读写，如果是git clone就不会有这个问题。

git pull origin master --allow-unrelated-histories

情景七：The authenticity of host ‘github.com (192.30.252.128)’ can’t be established.

  1. 这是因为没有known_hosts的原因，直接写yes就可以了。known_hosts主要是用来记录访问过的计算机。
