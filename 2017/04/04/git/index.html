情景五：如果想要删除本次修改的内容怎么办？

1.本次修改还没git add 这时候可以用git checkout –file就可以了
2.本次已经add或者commit但是没有推送到远程 这时候可以返回之前的版本

  • 2.1如果是add没有commit，这时候git reflow 选择最新的版本的commit Id 然后git reset –hard xx
  • 2.2如果commit但是没有推送到远程，这时候可以用也是和上一个方法一样，但是选择的commit id应该是上一个的，而不是最新的

其他

1 . 在使用git log的时候如果想要输出的结果在一行，可以添加后缀–pretty=oneline。命令为 git log –pretty=oneline
2 . git status 可以查看当前有没有内容需要还没有add，有没有内容还没commit，建议经常用
3 . git diff比较工作区和暂存区之间的差别也就是说没有add和add之后的区别，也可以用来比较2个分支之间的差别，假如当前分支为master，如果想要比较和bugs上的区别，可以用git diff bugs
4 . 解决冲突的时候如果想要看图形界面可以用git log –graph
5 . 删除文件可以用rm file
6 . clone指定分支 git clone -b [branch-name] git@github.com:mcfly001/electron.git
7 . 删除远程 git remote rm origin
8 . 查看分支(本地和远程) git branch -a
9 . git远程和本地的分支名字要保持一致。

git日常使用命令

git clone -b //从远程克隆一份指定的分支，git clone -b 分支名字
git diff –cached //这个查看的是add了，但是没有commit的修改的内容。
git checkout -b “name” //创建+切换分支
git branch -d “name” //删除分支
git branch -r //查看远程分支
git stash //保存当前的工作状态
git remote -v //查看远程的库
git remote add origin git@github.com:mcfly001/repositoryname.git //本地和github建立联系
git push origin “branchname” //将本地分支传递给远程
git push origin –d //删除远程分支
git remote rm ‘name’ //删除远程库
